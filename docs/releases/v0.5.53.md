# Release v0.5.53

## Summary

This release turns Relay's admin shell into a more governable collaboration system with better status controls, group membership operations, workflow run lifecycle APIs, and file auditability.

## Delivered

- expanded `PATCH /api/v1/users/:id/status`
- `GET /api/v1/user-groups/mentions`
- `GET /api/v1/user-groups/:id/members`
- `POST /api/v1/user-groups/:id/members`
- `DELETE /api/v1/user-groups/:id/members/:userId`
- `GET /api/v1/workflows/runs/:id`
- `POST /api/v1/workflows/runs/:id/cancel`
- `POST /api/v1/workflows/runs/:id/retry`
- `PATCH /api/v1/files/:id/retention`
- `GET /api/v1/files/:id/audit`

## Why It Matters

Relay now supports more realistic team operations:

- people can set richer custom statuses
- user groups can be managed as real mentionable collections
- files have retention and audit context instead of a binary archive/delete model
- workflows can be inspected, cancelled, and retried from the product shell
