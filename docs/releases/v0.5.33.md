# Release v0.5.33

## Summary

This release hardens the handoff between Gemini's latest AI/canvas UI work and the backend by fixing the `new-doc` artifact sentinel flow and adding explicit slash-command forwarding for AI execution.

## Delivered

- `POST /api/v1/ai/execute` support for optional `command`
- frontend-safe handling for local `new-doc` canvas creation
- real artifact creation when a new canvas is first saved
- artifact diff response mapping aligned with the backend envelope

## Why It Matters

This removes noisy 404s from the browser, makes slash-command intent visible to the backend, and keeps the AI/canvas workflow coherent while the product continues moving toward deeper agentic orchestration.

## Verification

- `cd apps/api && go test ./...`
- `cd apps/api && GOCACHE=$(pwd)/.cache/go-build go build ./...`
- `pnpm build`
