# v0.6.67 - Phase 72 WorkspaceView Contract Hotfix

This hotfix closes the remaining WorkspaceView API contract gaps after Gemini's backend work and Windsurf's Web consumption pass.

## Fixed

- `GET /api/v1/workspace/views` now returns `next_cursor`.
- `GET /api/v1/workspace/views?cursor=...` now honors cursor pagination.
- `PATCH /api/v1/workspace/views/:id` now supports validated `source` and `view_type` updates.
- Create/update now reject empty `title` and `source` values.
- Create/update now reject nested `filters` and nested `actions` descriptors.
- Unknown shallow action types remain metadata-only and are preserved.
