# v0.5.86 - Phase 46 Knowledge Entities and Wiki Substrate

This release adds first-class knowledge entities to Relay Agent Workspace. Entities are the durable wiki-style objects that connect files, messages, artifacts, workflows, and future business-domain data.

## Added

- `GET /api/v1/knowledge/entities`
- `POST /api/v1/knowledge/entities`
- `GET /api/v1/knowledge/entities/:id`
- `PATCH /api/v1/knowledge/entities/:id`
- `GET /api/v1/knowledge/entities/:id/refs`
- `POST /api/v1/knowledge/entities/:id/refs`
- `GET /api/v1/knowledge/entities/:id/timeline`
- `POST /api/v1/knowledge/entities/:id/events`
- `GET /api/v1/knowledge/entities/:id/links`
- `POST /api/v1/knowledge/links`
- `GET /api/v1/knowledge/entities/:id/graph`

## Data Model

- `KnowledgeEntity`
- `KnowledgeEntityRef`
- `KnowledgeEntityLink`
- `KnowledgeEvent`

## Contract Notes

- Citation lookup now hydrates `entity_title` when `entity_id` is linked to a `KnowledgeEntity`.
- Graph preview is generated from explicit refs and links; no graph database is introduced.
- Live external event ingestion remains Phase 47.

## Windsurf Handoff

- Add knowledge/entity list and detail pages.
- Use refs and timeline APIs to build the first wiki substrate.
- Use graph preview for relationship badges or a lightweight relationship panel.
- Update citation cards so `entity_id` links to entity details and `entity_title` is shown when present.
