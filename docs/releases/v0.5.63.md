# Relay Agent Workspace v0.5.63

This release aligns the Phase 33 backend contracts with Gemini's integrated frontend stores.

## Highlights

- channel-first list creation compatibility
- UI-friendly aliases for lists, list items, tool runs, and virtual/template artifacts
- tool run filtering by `channel_id`
- updated collaboration state showing Phase 33 integration as complete

## Added / Hardened Contracts

- `POST /api/v1/lists` can now derive `workspace_id` from `channel_id`
- `GET /api/v1/lists`
- `GET /api/v1/lists/:id`
  now include:
  - `user_id`
- `POST /api/v1/lists/:id/items`
- `PATCH /api/v1/lists/:id/items/:itemId`
- `GET /api/v1/lists/:id`
  item payloads now include:
  - `list_id`
  - `user_id`
- `GET /api/v1/tools/runs`
  now supports:
  - `channel_id`
- `GET /api/v1/tools/runs`
- `GET /api/v1/tools/runs/:id`
- `POST /api/v1/tools/:id/execute`
  now include:
  - `user_id`
  - `channel_id`
  - `finished_at`
  - `duration_ms`
- virtual `GET /api/v1/artifacts/new-doc`
- `POST /api/v1/artifacts/from-template`
  now include:
  - `user_id`
