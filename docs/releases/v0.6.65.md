# v0.6.65 - Phase 72 Super App Home Contract

This release freezes the Phase 72 collaboration contract for the next Slack-like workspace upgrade. It does not ship the full Home/Web/API implementation yet; it gives Gemini and Windsurf a deterministic execution plan.

## Scope

- Rebuild Home below the existing Workspace Overview into a Slack-like workbench centered on Today and My Work.
- Add a shallow WorkspaceView registry for future list, calendar, search, report, form, and channel-message surfaces.
- Keep WorkspaceView actions metadata-only in this phase.
- Prevent Phase 72 from turning into a full Super App runtime, app builder, BI engine, or Calendar/Reports/Forms product build.

## Contract Highlights

- Home section limits are frozen: Today 8, My Work 8, Recent Channels 6, AI Suggestions 5, Activity 10.
- Home deterministic ownership is required: the same logical item ID must not appear in more than one of Today, My Work, and Activity.
- WorkspaceView APIs are frozen at `/api/v1/workspace/views` and `/api/v1/workspace/views/:id`.
- WorkspaceView `source` must persist and be queryable.
- Web must tolerate unknown future `view_type` values by rendering a generic registry entry.
- Home core sections must still render if WorkspaceView or Apps & Tools loading fails.

## Handoff

- Gemini owns backend/API/tests for Home aggregation and WorkspaceView registry.
- Windsurf owns Web/UI for the Home workbench, Apps & Tools, WorkspaceView rendering, and route safety.
- Codex owns contract questions, sequencing, final audit, collaboration docs, and release closeout.
