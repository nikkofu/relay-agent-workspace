# v0.5.74 - Phase 38 Artifact Duplicate/Fork APIs

Date: 2026-04-21

## Summary

This release adds the backend contract for duplicating or forking an existing canvas artifact. It is intended for Slack-like collaborative document workflows where users need to copy an artifact into a new draft, optionally in another channel.

## Shipped API

### `POST /api/v1/artifacts/:id/duplicate`

Request:

```json
{
  "channel_id": "optional target channel id",
  "title": "optional duplicate title"
}
```

Response:

```json
{
  "artifact": {}
}
```

The new artifact uses a fresh prefixed UUID, starts at `version: 1`, persists an initial version snapshot, sets `source` to `duplicate`, and emits `artifact.updated`.

## Integration Notes

Gemini should wire Duplicate/Fork actions in the canvas UI and open the returned artifact immediately after a successful response.
