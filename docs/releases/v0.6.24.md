# Relay Agent Workspace v0.6.24

Release date: 2026-04-23

## Summary

Phase 63G makes AI compose co-drafting activity durable. The UI can now hydrate recent AI suggestion activity after refresh instead of relying only on per-tab websocket memory.

## Added API

### `GET /api/v1/ai/compose/activity`

Returns newest-first compose activity rows:

```json
{
  "items": [
    {
      "id": "compose-activity-...",
      "compose_id": "compose-...",
      "workspace_id": "ws-1",
      "channel_id": "ch-1",
      "dm_id": "",
      "thread_id": "",
      "intent": "reply",
      "suggestion_count": 2,
      "provider": "openrouter",
      "model": "nvidia/nemotron-3-super-120b-a12b:free",
      "created_at": "2026-04-23T10:00:00Z"
    }
  ]
}
```

Supported query filters:

- `channel_id`
- `dm_id`
- `workspace_id`
- `intent`
- `limit`, capped at 100

## Persistence

Successful sync compose and finalized streaming compose now persist an `AIComposeActivity` row with:

- `compose_id`
- `workspace_id`
- `channel_id` or `dm_id`
- `thread_id`
- `intent`
- `suggestion_count`
- `provider`
- `model`
- `created_at`

## Realtime

`knowledge.compose.suggestion.generated` now includes:

- `compose`
- `activity`

## Windsurf Handoff

- Hydrate `composeSuggestionActivity` from `GET /api/v1/ai/compose/activity?channel_id=...&limit=50`.
- Continue appending live websocket events, preferring `payload.activity`.
- Consider rendering the activity in `#agent-collab`, channel info, or workspace Activity.
