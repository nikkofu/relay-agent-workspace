# v0.6.6 - Phase 60 Knowledge Distribution APIs

This release extends Relay's knowledge layer from operational controls into distribution and broadcast. The backend can now summarize followed-entity state, generate shareable entity links, and push trending rerank events live to connected clients.

## Added APIs

- `GET /api/v1/users/me/knowledge/followed/stats`
- `POST /api/v1/knowledge/entities/:id/share`

## Added Websocket Event

- `knowledge.trending.changed`

## Behavior

- Followed stats return `total_count`, `spiking_count`, `muted_count`, and `by_kind[]`.
- Entity sharing returns:
  - `url`
  - `short_url`
  - `relative_path`
- Trending rerank events are emitted when new knowledge refs are created directly or through auto-link flows, using the same ranking model as `GET /api/v1/knowledge/trending`.

## Windsurf Handoff

- Add a summary strip to Following Hub using `GET /api/v1/users/me/knowledge/followed/stats`.
- Add share actions to Trending cards and entity detail headers using `POST /api/v1/knowledge/entities/:id/share`.
- Listen for `knowledge.trending.changed` to live-refresh Knowledge and Home trending cards.
