# Relay Agent Workspace v0.6.26

Release date: 2026-04-23

## Summary

Phase 63H adds the next AI-native backend automation layer for Relay:

- compose activity can now be aggregated, not just listed
- stale entity briefs can move through a durable automation lifecycle
- schedule intent can become a real booking record with ICS fallback

## Added API

### `GET /api/v1/ai/compose/activity/digest`

Returns aggregated compose activity across workspace, channel, or DM scope.

Supported query params:

- `workspace_id`
- `channel_id`
- `dm_id`
- `window=1h|24h|7d|custom`
- `start_at`
- `end_at`
- `intent`
- `group_by=intent|channel|user|provider|model`
- `limit`

Notes:

- `AIComposeActivity` now persists `user_id` for new rows.
- historical null-user rows stay in totals, map to `unknown` only for `group_by=user`, and do not count toward `summary.unique_users`.

### Entity brief automation

- `GET /api/v1/knowledge/entities/:id/brief/automation`
- `POST /api/v1/knowledge/entities/:id/brief/automation/run`
- `POST /api/v1/knowledge/entities/:id/brief/automation/retry`

Adds durable `AIAutomationJob` state for `entity_brief_regen` plus queue/retry semantics.

### Schedule booking

- `POST /api/v1/ai/schedule/book`
- `GET /api/v1/ai/schedule/bookings`
- `GET /api/v1/ai/schedule/bookings/:id`
- `POST /api/v1/ai/schedule/bookings/:id/cancel`

Adds durable `AIScheduleBooking` records with:

- `requested_by`
- `compose_id`
- `provider`
- `status`
- `ics_content`

## Realtime

New websocket events:

- `knowledge.entity.brief.regen.queued`
- `knowledge.entity.brief.regen.started`
- `knowledge.entity.brief.regen.failed`
- `schedule.event.booked`
- `schedule.event.cancelled`

Existing success event remains:

- `knowledge.entity.brief.generated`

## Windsurf Handoff

- Build a workspace/channel AI analytics strip from `GET /api/v1/ai/compose/activity/digest`.
- Hydrate entity detail automation state from `GET /knowledge/entities/:id/brief/automation`.
- Wire schedule chips to `POST /api/v1/ai/schedule/book` and subscribe to `schedule.event.booked` / `schedule.event.cancelled`.
- Treat `booking.ics_content` as the durable fallback artifact even before Google/Outlook adapters exist.
