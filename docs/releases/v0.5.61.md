# Relay Agent Workspace v0.5.61

This release starts the next Slack-parity wave by introducing structured work objects across lists, tool execution, and canvas templating.

## Highlights

- shared workspace lists and checklist item APIs
- tool execution run history and detail payloads
- artifact templates plus virtual `new-doc` bootstrap support
- seeded runtime data for lists and tool runs

## Added APIs

- `GET /api/v1/lists`
- `POST /api/v1/lists`
- `GET /api/v1/lists/:id`
- `PATCH /api/v1/lists/:id`
- `DELETE /api/v1/lists/:id`
- `POST /api/v1/lists/:id/items`
- `PATCH /api/v1/lists/:id/items/:itemId`
- `DELETE /api/v1/lists/:id/items/:itemId`
- `GET /api/v1/tools/runs`
- `GET /api/v1/tools/runs/:id`
- `POST /api/v1/tools/:id/execute`
- `GET /api/v1/artifacts/templates`
- `POST /api/v1/artifacts/from-template`
- virtual `GET /api/v1/artifacts/new-doc`

## Gemini Integration Notes

- use list APIs for checklist panels in Home, channel sidebars, or a dedicated Lists surface
- use tool-run APIs for activity cards, execution inspectors, and tool launch actions
- use artifact templates and `new-doc` virtual payloads to replace frontend-only blank canvas assumptions
