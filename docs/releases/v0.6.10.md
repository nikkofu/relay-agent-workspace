# v0.6.10 - Phase 62 Cached Brief And Bulk Read APIs

Release date: 2026-04-22

## Added By Codex

- `GET /api/v1/knowledge/entities/:id/brief`
  - Returns cached entity brief content from `AISummary`.
  - Does not invoke the LLM gateway.
  - Returns `brief: null` when no cached brief exists.
- Websocket `knowledge.entity.brief.generated`
  - Broadcast after `POST /api/v1/knowledge/entities/:id/brief` successfully persists a generated brief.
- `GET /api/v1/knowledge/weekly-brief?workspace_id=...`
  - Returns cached per-user weekly knowledge brief content.
  - Does not invoke the LLM gateway.
  - Returns `brief: null` when no cached weekly brief exists.
- `POST /api/v1/notifications/bulk-read`
  - Accepts `item_ids[]`.
  - De-duplicates IDs.
  - Persists read markers in one transaction.
  - Broadcasts websocket `notifications.bulk_read`.

## Windsurf Handoff

- Hydrate entity brief UI with `GET /knowledge/entities/:id/brief` on page load.
- Keep `POST /knowledge/entities/:id/brief` for Generate/Regenerate only.
- Listen for `knowledge.entity.brief.generated` to update entity detail across tabs.
- Hydrate weekly digest UI with `GET /knowledge/weekly-brief?workspace_id=...`.
- Replace per-item mark-all-read loops with `POST /notifications/bulk-read`.
