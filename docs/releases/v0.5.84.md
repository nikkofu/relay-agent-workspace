# v0.5.84 - Phase 44 File Extraction, Search, and Citation APIs

This release upgrades files into indexed knowledge objects. Relay can now extract content from common document formats, persist chunk-level search data, surface explicit indexing state, and expose citation-ready file chunks for later AI workflows.

## Added

- `GET /api/v1/files/:id/extraction`
- `POST /api/v1/files/:id/extraction/rebuild`
- `GET /api/v1/files/:id/extracted-content`
- `GET /api/v1/files/:id/chunks`
- `GET /api/v1/files/:id/citations`
- `GET /api/v1/search/files?q=...`
- websocket `file.extraction.updated`
- `apps/api/internal/fileindex/`

## Extraction Coverage

- real extraction:
  - `txt`
  - `md`
  - `pdf`
  - `docx`
  - `xlsx`
  - `pptx`
- mock OCR path:
  - `png`
  - `jpg`
  - `jpeg`
  - `webp`
- explicit unsupported legacy formats:
  - `doc`
  - `xls`
  - `ppt`

## Contract Notes

- Raw file download remains `GET /api/v1/files/:id/content`
- Extracted text lives at `GET /api/v1/files/:id/extracted-content`
- File payloads now include indexing and OCR metadata for UI state rendering
