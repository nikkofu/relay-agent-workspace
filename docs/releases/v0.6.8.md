# v0.6.8 - Phase 61 AI Knowledge Brief And Presence APIs

Release date: 2026-04-22

## Added By Codex

- `POST /api/v1/knowledge/entities/:id/brief`
  - Generates an LLM-backed entity brief from entity metadata, recent refs, and timeline events.
  - Supports `provider`, `model`, and `force`.
  - Persists cached output in `AISummary` scope `knowledge_entity`.
- `POST /api/v1/knowledge/weekly-brief`
  - Generates a per-user weekly followed-knowledge brief from followed stats, followed entities, and trending entities.
  - Supports `workspace_id`, `provider`, `model`, and `force`.
  - Persists cached output in `AISummary` scope `knowledge_weekly`.
- `GET /api/v1/knowledge/entities/:id/activity/backfill-status`
  - Reports historical ref coverage for an entity.
- `POST /api/v1/knowledge/entities/:id/activity/backfill`
  - Scans historical messages and files for entity-title matches.
  - Creates missing `KnowledgeEntityRef` rows.
  - Emits `knowledge.entity.ref.created` and `knowledge.trending.changed` when new refs are created.
- `GET /api/v1/presence/bulk`
  - Supports optional `channel_id`.
  - Returns hydrated users plus aggregate presence counts.
- Websocket `knowledge.followed.stats.changed`
  - Emitted after follow/unfollow, per-follow notification updates, and bulk follow notification updates.

## Windsurf Handoff

- Add entity-detail brief generation/regeneration using `POST /knowledge/entities/:id/brief`.
- Add a weekly followed-knowledge digest CTA using `POST /knowledge/weekly-brief`.
- Display backfill status on entity activity/sparkline surfaces and expose a dev/admin backfill action.
- Listen for `knowledge.followed.stats.changed` to update the Following Hub stats strip without manual refetch coupling.
- Use `GET /presence/bulk?channel_id=...` for reconnect and channel-switch presence hydration.
