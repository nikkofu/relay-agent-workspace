# Relay Agent Workspace v0.6.36

## Summary

This release implements Phase 65A: durable user mention semantics across message creation, unified activity, mentions, and inbox.

Relay now treats `@user` mentions as a real backend primitive instead of relying on legacy message-content scanning.

## API Coverage

- Channel message creation now parses explicit `@user` mentions and persists:
  - `message.metadata.user_mentions[]`
  - durable `MessageMention` rows
- DM message creation now parses explicit `@user` mentions and persists durable `MessageMention` rows
- `GET /api/v1/activity/feed`
  - `mention` rows now support:
    - `mention_kind=user`
    - `mention_kind=entity`
  - user mention rows include:
    - `meta.message_id`
    - `meta.mentioned_user_id`
    - `meta.mentioned_by_user_id`
- `GET /api/v1/mentions`
  - now queries persisted `MessageMention`
- `GET /api/v1/inbox`
  - mention branch now uses the same persisted mention source
- Realtime:
  - `mention.created`

## Notes

- Self-mentions are persisted but excluded from the current user's mention-facing query surfaces.
- DM mention rows now resolve to DM-aware deep links.
- This release is the first concrete continuation of the remembered follow-up `2`: keep strengthening the unified feed itself.

## Windsurf Handoff

- Treat `mention_kind=user|entity` as stable backend semantics.
- Use `meta.mentioned_user_id` for user-mention specific UI such as badges, filters, or future Mentions-only tabs powered from the unified feed.
- Subscribe to `mention.created` for realtime append in Activity, Inbox, and Mentions surfaces.
- Expect DM mention rows to deep-link to `/workspace/dms?id=<dm_id>`.
