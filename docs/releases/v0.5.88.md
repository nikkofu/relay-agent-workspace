# v0.5.88 - Phase 48 Channel Knowledge Context

This release connects active Slack-like channels to Relay's AI-native knowledge layer. Channel views can now ask the backend for the most recent knowledge refs in that channel, and citation lookup can hydrate entity links directly from the canonical `KnowledgeEntityRef` table.

## Added

- `GET /api/v1/channels/:id/knowledge`
- Channel knowledge ref payload fields:
  - `entity_id`
  - `entity_title`
  - `entity_kind`
  - `ref_kind`
  - `ref_id`
  - `role`
  - `source_title`
  - `source_snippet`
  - `created_at`

## Changed

- `GET /api/v1/citations/lookup` now hydrates `entity_id` and `entity_title` from `KnowledgeEntityRef` message/file associations.
- Auto-linked messages and files are now visible as entity-aware citation results without requiring separate evidence-link records.
- Channel knowledge context returns message and file refs sorted newest-first with `limit` support.

## Windsurf Handoff

- Build a channel knowledge banner or right-side panel using `GET /api/v1/channels/:id/knowledge`.
- Refresh the channel context when `knowledge.entity.ref.created` arrives for the active channel.
- Let `CitationCard` rely on hydrated `entity_id/entity_title` from citation lookup and link directly to `/workspace/knowledge/[entity_id]`.
