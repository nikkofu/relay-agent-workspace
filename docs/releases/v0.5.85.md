# v0.5.85 - Phase 45 AI Citation Lookup APIs

This release adds a unified citation lookup layer for Relay Agent Workspace. Evidence can now be searched across file chunks, channel messages, thread messages, and artifact sections through one backend contract, while reserving entity-aware fields for the upcoming wiki and graph phases.

## Added

- `GET /api/v1/citations/lookup?q=...`
- `apps/api/internal/knowledge/`
- `KnowledgeEvidenceLink`
- `KnowledgeEvidenceEntityRef`

## Evidence Coverage

- `file_chunk`
- `message`
- `thread`
- `artifact_section`

## Contract Notes

- Citation payloads now converge on:
  - `id`
  - `evidence_kind`
  - `source_kind`
  - `source_ref`
  - `ref_kind`
  - `locator`
  - `snippet`
  - `title`
  - `score`
  - `entity_id`
  - `entity_title`
- `GET /api/v1/files/:id/citations` now follows the same shared citation shape.
- `POST /api/v1/ai/execute` accepts a `citations` field for future execution-time grounding.

## Windsurf Handoff

- Build a shared citation card and citation inspector around `evidence_kind`.
- Consume `GET /api/v1/citations/lookup` for mixed evidence search.
- Reuse the same renderer for `GET /api/v1/files/:id/citations`.
- Treat `entity_title` as optional in this release; Phase 46 will enrich entity-level wiki data.
