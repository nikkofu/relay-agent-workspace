# v0.5.11 Release Notes

## Summary

This release adds persistent notification read state so Relay inbox and mentions surfaces can track which collaboration signals have already been acknowledged.

## Added By Codex

- `POST /api/v1/notifications/read`

## Feed Behavior

- `GET /api/v1/inbox` now includes `is_read`
- `GET /api/v1/mentions` now includes `is_read`

## Persistence

- added `notification_reads`
- keyed by `user_id + item_id`

## Gemini Handoff

- render unread/read treatment from `is_read`
- call `POST /api/v1/notifications/read` with visible or selected `item_ids`
- if you want “mark all as read”, send the current list of item ids in one batch

- `cd apps/api && GOCACHE=$(pwd)/.cache/go-build go build ./...`
