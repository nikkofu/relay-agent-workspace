# v0.5.99

Released: 2026-04-22

## Summary

Phase 56 closes the remaining backend gaps after Windsurf's `v0.5.98` knowledge UI pass. Relay now supports:

- digest inbox drill-down via `GET /api/v1/knowledge/inbox/:id`
- digest schedule dry-run preview via `POST /api/v1/channels/:id/knowledge/digest/preview-schedule`
- cross-device settings hydration via `GET /api/v1/me/settings`
- partial settings persistence for AI + appearance preferences via `PATCH /api/v1/me/settings`

## API Additions

### `GET /api/v1/knowledge/inbox/:id`

Returns a single knowledge inbox item plus rich drill-down context:

- `id`
- `channel`
- `message`
- `digest`
- `is_read`
- `occurred_at`
- `entity_contexts[]`

Each `entity_contexts[]` item includes:

- `entity_id`
- `entity_title`
- `entity_kind`
- `delta`
- `messages[]` representative source messages for that entity in the digest channel

### `POST /api/v1/channels/:id/knowledge/digest/preview-schedule`

Accepts the same cadence payload as schedule save, plus optional `count`.

Request fields:

- `window`
- `timezone`
- `day_of_week`
- `day_of_month`
- `hour`
- `minute`
- `limit`
- `pin`
- `is_enabled`
- `count`

Response fields:

- `schedule`
- `upcoming_runs[]`
- `digest`

This is intended for live dialog preview before persisting the schedule.

### `GET /api/v1/me/settings`

Returns:

- `provider`
- `model`
- `mode`
- `theme`
- `message_density`
- `locale`
- `timezone`

## Behavior Changes

- `PATCH /api/v1/me/settings` now supports partial updates instead of full overwrite.
- New persisted user preference fields:
  - `theme_preference`
  - `message_density`
  - `locale`
- `theme` validation accepts `light | dark | system`
- `message_density` validation accepts `comfortable | compact`
- `timezone` validation now rejects invalid IANA timezone identifiers

## Windsurf Handoff

- hydrate `/workspace/settings` from `GET /api/v1/me/settings`
- persist Appearance tab changes through `PATCH /api/v1/me/settings`
- load selected inbox items through `GET /api/v1/knowledge/inbox/:id`
- call `POST /api/v1/channels/:id/knowledge/digest/preview-schedule` from the digest schedule dialog while editing cadence/timezone

## Remaining Recommended Backend Target

- websocket `knowledge.entity.activity.spiked`
