# v0.5.87 - Phase 47 Knowledge Live Events and Auto-Linking

This release turns the knowledge wiki substrate into a live collaboration surface. Entity pages can now update through WebSocket events, ingest external/domain events, show richer graph semantics, and receive deterministic refs from new messages and uploaded files.

## Added

- `POST /api/v1/knowledge/events/ingest`
- WebSocket `knowledge.entity.created`
- WebSocket `knowledge.entity.updated`
- WebSocket `knowledge.entity.ref.created`
- WebSocket `knowledge.event.created`
- WebSocket `knowledge.link.created`
- Deterministic message-to-entity auto-linking with `ref_kind=message` and `role=discussion`
- Deterministic file-to-entity auto-linking with `ref_kind=file` and `role=evidence`
- Graph edge metadata: `weight`, `direction`, `role`
- Graph ref-node metadata: `source_kind`, `ref_kind`, `ref_id`, `role`

## Contract Notes

- `POST /api/v1/knowledge/events/ingest` accepts `entity_id`, `event_type`, `title`, `body`, `actor_user_id`, `source_kind`, and `source_ref`.
- Message creation still broadcasts `message.created` before any derived `knowledge.entity.ref.created` event.
- File upload still broadcasts `file.extraction.updated` before any derived `knowledge.entity.ref.created` event.
- Auto-linking is deterministic title matching for now. LLM extraction and confidence scoring remain future agentic work.
- Graph preview remains relational SQLite-backed; no graph database is introduced.
