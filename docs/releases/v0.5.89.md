# v0.5.89 - Phase 49 Channel Knowledge Summary And Entity Mentions

This release extends Relay's channel knowledge layer from a raw ref feed into an aggregate view that the UI can render directly. It also adds the first dedicated backend contract for knowledge-entity mentions in the message composer.

## Added

- `GET /api/v1/channels/:id/knowledge/summary`
- Query params:
  - `limit`
  - `days`
- Summary payload fields:
  - `channel_id`
  - `window_days`
  - `total_refs`
  - `recent_ref_count`
  - `top_entities`
- Each `top_entities[]` entry includes:
  - `entity_id`
  - `entity_title`
  - `entity_kind`
  - `ref_count`
  - `message_ref_count`
  - `file_ref_count`
  - `last_ref_at`
  - `trend`
- `GET /api/v1/knowledge/entities/suggest`
- Suggest query params:
  - `q`
  - `channel_id`
  - `workspace_id`
  - `limit`
- Suggestion payload fields:
  - `id`
  - `title`
  - `kind`
  - `summary`
  - `source_kind`
  - `ref_count`
  - `channel_ref_count`

## Changed

- Channel knowledge can now be consumed in two modes:
  - newest refs feed via `GET /api/v1/channels/:id/knowledge`
  - aggregated entity summary via `GET /api/v1/channels/:id/knowledge/summary`
- Knowledge entity autocomplete now scopes to the active channel's workspace and ranks by channel-local reference activity.

## Windsurf Handoff

- Build a compact channel knowledge summary card from `GET /api/v1/channels/:id/knowledge/summary?days=7&limit=5`.
- Use `GET /api/v1/knowledge/entities/suggest?q=...&channel_id=...&limit=8` for `@entity:` composer autocomplete.
- On `knowledge.entity.ref.created` for the active channel, refresh both channel knowledge endpoints and show a lightweight toast/banner for the newest linked entity.
