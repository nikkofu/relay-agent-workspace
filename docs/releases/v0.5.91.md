# v0.5.91 - Phase 51 Knowledge Discovery APIs

## Summary

This release expands Relay's AI-native knowledge layer from passive context into active discovery. The backend now supports:

- entity-centric message search
- live entity hover summaries
- daily/weekly/monthly channel knowledge digests
- publishing digests as pinned channel messages with structured metadata

## Added APIs

### Entity Message Discovery

- `GET /api/v1/search/messages/by-entity`
- Query params:
  - `entity_id` required
  - `channel_id` optional
  - `limit` optional
- Response fields per message:
  - `id`
  - `channel_id`
  - `user_id`
  - `content`
  - `thread_id`
  - `created_at`
  - `metadata`
  - `snippet`
  - `entity_id`
  - `entity_title`
  - `match_sources`

### Entity Hover Summary

- `GET /api/v1/knowledge/entities/:id/hover`
- Query params:
  - `channel_id` optional
  - `days` optional
- Response fields:
  - `ref_count`
  - `channel_ref_count`
  - `message_ref_count`
  - `file_ref_count`
  - `recent_ref_count`
  - `last_activity_at`
  - `related_channels[]`

### Channel Knowledge Digest

- `GET /api/v1/channels/:id/knowledge/digest`
- Query params:
  - `window` = `daily` | `weekly` | `monthly`
  - `limit`
- Response fields:
  - `channel_id`
  - `window`
  - `window_days`
  - `generated_at`
  - `total_refs`
  - `recent_ref_count`
  - `headline`
  - `summary`
  - `top_movements[]`

### Publish Digest Message

- `POST /api/v1/channels/:id/knowledge/digest/publish`
- Request body:
  ```json
  {
    "window": "weekly",
    "limit": 5,
    "pin": true
  }
  ```
- Returns:
  - `message`
  - `digest`

## Metadata Contract

Published digest messages now preserve:

- `message.metadata.knowledge_digest`

This is intentionally designed so the web layer can render a structured digest card from the message payload itself.

## Windsurf Handoff

Please wire the following next:

1. `EntityMentionChip` hover enrichment via `GET /api/v1/knowledge/entities/:id/hover?channel_id=...&days=7`
2. Entity drilldown results sheet via `GET /api/v1/search/messages/by-entity?entity_id=...&channel_id=...`
3. Channel digest preview banner via `GET /api/v1/channels/:id/knowledge/digest?window=weekly&limit=5`
4. Digest publish CTA via `POST /api/v1/channels/:id/knowledge/digest/publish`

Important note:

- Published digest messages do not auto-link themselves back into knowledge refs. That avoids recursive digest noise.
