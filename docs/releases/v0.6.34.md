# Relay Agent Workspace v0.6.34

## Summary

This release completes the backend side of Phase 64C for the unified activity rail.

`GET /api/v1/activity/feed` now covers the remaining core Slack-like event families needed by Windsurf's shared activity UI:

- `artifact_updated`
- `tool_run`
- `reply`
- `dm_message`
- `mention`
- `reaction`

The endpoint now spans 12 event types from one REST surface, with stable deep links and lightweight `meta` payloads for each source family.

## API Coverage

- `artifact_updated`
  - source: persisted `Artifact`
  - meta: `artifact_id`, `artifact_type`, `version_id`
- `tool_run`
  - source: persisted `ToolRun`
  - meta: `run_id`, `tool_name`, `status`
- `reply`
  - source: persisted threaded channel `Message`
  - meta: `message_id`, `thread_id`
- `dm_message`
  - source: persisted `DMMessage`
  - meta: `message_id`
- `mention`
  - source: persisted `message.metadata.entity_mentions`
  - meta: `message_id`, `mention_kind`
- `reaction`
  - source: persisted `MessageReaction`
  - meta: `message_id`, `emoji`

## Notes

- `mention` is intentionally deterministic-first in this release and currently maps from persisted `entity_mentions`; broader generic mention fanout remains a future feed-strengthening item.
- This release preserves the existing `UnifiedActivityFeedItem` contract, so Windsurf does not need a store/type migration.
- The user explicitly asked to remember the next two continuations after this round:
  - keep strengthening the unified feed itself
  - then broaden into larger Slack-like capability work outside the feed slice
