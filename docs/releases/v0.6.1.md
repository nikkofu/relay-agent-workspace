# v0.6.1

Released: 2026-04-22

## Summary

This release combines Windsurf's `v0.6.0` UI completion pass with Codex Phase 57 backend work. The main backend addition is the shift from passive knowledge follows to configurable, realtime alerting.

## API Additions

### `PATCH /api/v1/users/me/knowledge/followed/:id`

Updates follow notification preferences.

Request:

- `notification_level`

Allowed values:

- `all`
- `digest_only`
- `silent`

Response:

- `follow`
- `is_following`

### Websocket `knowledge.entity.activity.spiked`

Broadcast when a followed entity crosses the spike threshold.

Payload:

- `entity`
- `user_ids`
- `channel_id`
- `recent_ref_count`
- `previous_ref_count`
- `delta`
- `related_channel_ids`
- `occurred_at`

Frontends should only surface the alert when the current user's ID appears in `user_ids`.

## Persistence Changes

`knowledge_entity_follows` now store:

- `notification_level`
- `last_alerted_at`

New follows default to `notification_level = "all"`.

## Included UI Work From Windsurf

- Settings page now hydrates and persists theme/density through backend settings APIs.
- Knowledge Inbox detail pane loads real digest detail from `GET /api/v1/knowledge/inbox/:id`.
- Digest schedule dialog previews cadence with `POST /api/v1/channels/:id/knowledge/digest/preview-schedule`.
- DMs landing page, workspace header actions, set-status dialog, scrolling fixes, and AI avatar normalization are included in this release line.
