# v0.6.18 Release Notes

Release date: 2026-04-23

## Summary

Codex Phase 63D expands the AI composer from channel/thread-only suggestions into a shared channel/thread/DM composer substrate. It also adds intent variants and feedback aggregation so Windsurf can build richer AI-native composer controls without changing the suggestion-card shape.

## API Changes

- `POST /api/v1/ai/compose`
  - Accepts exactly one scope: `channel_id` or `dm_id`.
  - Keeps optional `thread_id` for channel thread context.
  - Supports `intent`: `reply`, `summarize`, `followup`, `schedule`.
  - Adds `dm_id` to the response when the request targets a DM.
- `POST /api/v1/ai/compose/stream`
  - Accepts the same channel/thread/DM scope contract.
  - SSE `start` and `suggestion.done` payloads include `dm_id` when applicable.
- `POST /api/v1/ai/compose/:id/feedback`
  - Accepts either `channel_id` or `dm_id`.
  - Persists the selected scope with `up`, `down`, or `edited` feedback.
- `GET /api/v1/ai/compose/:id/feedback/summary`
  - Returns aggregate counts and recent feedback rows for a suggestion ID.

## Windsurf Handoff

- Enable DM composer AI Suggest by sending `{ dm_id, draft, intent, limit }` to `POST /ai/compose/stream`.
- Add an intent selector for `reply`, `summarize`, `followup`, and `schedule`.
- Keep the existing fallback to sync `POST /ai/compose`.
- Use `GET /ai/compose/:id/feedback/summary` for a future Agent-Collab or admin learning-signal panel.
