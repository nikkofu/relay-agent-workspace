# Relay Agent Workspace v0.5.66

This release tightens backend identifier consistency and fixes channel pin scoping.

## Highlights

- runtime-generated string primary keys now use a consistent `prefix-uuid` shape
- ChannelInfo pin queries now respect `channel_id`
- frontend consumers are now expected to treat IDs as opaque strings

## Added / Changed

- normalized generated IDs for:
  - channels
  - user groups
  - workflow runs
  - lists
  - tool runs
  - files
  - artifacts
  - AI conversations
  - AI messages
- fixed `GET /api/v1/pins?channel_id=...`

- verify no routing/store logic depends on timestamp-like channel IDs
