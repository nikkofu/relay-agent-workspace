# v0.5.93 - Phase 53 Channel Persistence Hardening

## Summary

This release fixes the channel refresh-loss bug reported with `#game`.

Root cause:

- The frontend attempted to infer `workspaceId` from the first channel in local state.
- Backend channel payloads expose `workspace_id`, not `workspaceId`.
- That made `workspaceId` undefined and triggered the old mock fallback `ws_1`.
- The backend accepted `ws_1`, creating an orphan channel that appeared locally after creation but disappeared after refresh because the sidebar reloads `GET /api/v1/channels?workspace_id=ws-1`.

## Fixed

- `channel-store` now uses `workspace-store.currentWorkspace.id` for channel creation.
- `channel-store` maps backend snake_case channel payloads into frontend camelCase fields.
- `POST /api/v1/channels` now rejects unknown `workspace_id` values.
- API startup repairs legacy `workspace_id = "ws_1"` channel rows by moving them to `ws-1` when the Relay workspace exists.
- Empty duplicate legacy channels with the same name are cleaned up during repair to avoid duplicate sidebar entries.
