# v0.5.97 - Knowledge Follow And Composer Match APIs

## Summary

This release delivers the next knowledge-collaboration backend slice for Relay Agent Workspace.

The API now supports:

- per-user knowledge entity follow/unfollow
- followed-entity listing for profile or sidebar surfaces
- passive composer reverse lookup for existing knowledge entities in draft text

## Added

### Knowledge Follow APIs

- `GET /api/v1/users/me/knowledge/followed`
- `POST /api/v1/knowledge/entities/:id/follow`
- `DELETE /api/v1/knowledge/entities/:id/follow`

`GET /api/v1/users/me/knowledge/followed` returns `items[]`, where each item includes:

- `follow`
- `entity`

Follow creation is idempotent for the same `(entity_id, user_id)` pair.

### Composer Reverse Lookup API

- `POST /api/v1/knowledge/entities/match-text`

Request body:

```json
{
  "workspace_id": "ws-1",
  "text": "Launch Program needs a decision",
  "limit": 5
}
```

Response shape:

```json
{
  "matches": [
    {
      "entity_id": "entity-1",
      "entity_title": "Launch Program",
      "entity_kind": "project",
      "source_kind": "manual",
      "matched_text": "Launch Program",
      "start": 0,
      "end": 14
    }
  ]
}
```

Matching is deterministic, workspace-scoped, excludes archived entities, and prefers the longest non-overlapping entity titles.
