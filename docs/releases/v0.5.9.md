# v0.5.9 Release Notes

## Summary

This release adds the first starred and pinned discovery APIs so Relay can surface saved channels and pinned references as dedicated collaboration destinations.

## Added By Codex

- `GET /api/v1/starred`
- `POST /api/v1/channels/:id/star`
- `GET /api/v1/pins`

## Behavior

- starred channels are returned from persisted `channels.is_starred`
- toggling a channel star returns the updated channel and `is_starred`
- pins return hydrated message items with:
  - `message`
  - `channel`
  - `user`

## Persistence

- reused:
  - `channels.is_starred`
  - `messages.is_pinned`
- seeded one pinned message for local development visibility

## Gemini Handoff

- use `GET /api/v1/starred` for dedicated starred surfaces instead of filtering only the full channel list
- use `POST /api/v1/channels/:id/star` for explicit star/unstar actions
- use `GET /api/v1/pins` for pinned message lists and navigation targets
