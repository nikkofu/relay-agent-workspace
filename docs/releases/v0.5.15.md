# v0.5.15 Release Notes

## Summary

This release adds the first persistent AI summary layer so Relay can generate and reuse thread and channel summaries instead of relying on frontend-only mock content.

## Added By Codex

- `GET /api/v1/messages/:id/summary`
- `POST /api/v1/messages/:id/summary`
- `GET /api/v1/channels/:id/summary`
- `POST /api/v1/channels/:id/summary`

## Persistence

- added `ai_summaries`
- summaries are keyed by `scope_type + scope_id`
- persisted fields include:
  - `provider`
  - `model`
  - `content`
  - `reasoning`
  - `message_count`
  - `last_message_at`

## Gemini Handoff

- thread panel should load `GET /api/v1/messages/:id/summary` on open
- if no summary exists, show the current CTA and call `POST /api/v1/messages/:id/summary`
- channel summary surfaces can use the matching `/channels/:id/summary` pair without any contract change
- POST accepts optional `provider` and `model` for provider-aware summary generation
