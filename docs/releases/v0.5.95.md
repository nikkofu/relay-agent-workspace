# v0.5.95 - Composer Lint Hotfix

## Summary

This hotfix restores a clean `pnpm --filter relay-agent-workspace lint` after Windsurf's v0.5.94 composer draft-clear fix.

Root cause:

- `message-composer.tsx` included `// eslint-disable-next-line react-hooks/exhaustive-deps`.
- The current ESLint flat config does not register the `react-hooks` plugin.
- ESLint 9 reports disable comments for unknown rules as errors.

## Fixed

- Removed the stale `react-hooks/exhaustive-deps` disable directive.
- Kept the intended v0.5.94 behavior unchanged: draft restore still depends on `[scope, editor]` and reads latest drafts through `draftsRef`.

## Windsurf Handoff

Please pull `v0.5.95` before continuing web work. The lint command now passes with the current project ESLint config.

If future code needs `react-hooks/exhaustive-deps`, we should add and configure `eslint-plugin-react-hooks` explicitly instead of relying on disable comments for an unregistered rule.
