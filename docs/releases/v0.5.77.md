# v0.5.77 - Phase 40 Dynamic Agent-Collab APIs

Date: 2026-04-21

## Summary

This release adds backend contracts for Windsurf's new #agent-collab hub. The collaboration page can now move away from static TypeScript data and consume Markdown-backed API data plus realtime sync.

## Shipped APIs

### `GET /api/v1/agent-collab/members`

Returns team members parsed from the `Member Profiles` table in `docs/AGENT-COLLAB.md`.

Response:

```json
{
  "members": [
    {
      "name": "Windsurf",
      "role": "Web/UI Agent",
      "specialty": "Component Architecture, TypeScript, UX Flows, Agent Collaboration UI",
      "primary_tools": "apps/web, write_file, multi_edit"
    }
  ]
}
```

### `POST /api/v1/agent-collab/comm-log`

Persists a new communication log entry back into `docs/AGENT-COLLAB.md`.

Request:

```json
{
  "from": "Codex",
  "to": "Windsurf",
  "title": "Phase 40 Dynamic Agent-Collab API",
  "content": "The dynamic hub API is ready for frontend integration."
}
```

Response:

```json
{
  "entry": {
    "id": "comm_...",
    "date": "2026-04-21",
    "title": "Phase 40 Dynamic Agent-Collab API",
    "from": "Codex",
    "to": "Windsurf",
    "content": "The dynamic hub API is ready for frontend integration.",
    "created_at": "2026-04-21T..."
  }
}
```

## Realtime Contract

`GET /api/v1/agent-collab/snapshot` and websocket `agent_collab.sync` now include:

- `active_superpowers`
- `comm_log`
- `members`
- `task_board`

Creating a comm-log entry broadcasts `agent_collab.sync` on `ch-collab`.

## Windsurf Handoff

Replace static `MEMBERS` and `COMM_SECTIONS` usage in `apps/web/components/agent-collab/` with the new snapshot/API payloads. Keep the static module as fallback data only.
