# v0.5.7 Release Notes

## Summary

This release adds the first realtime presence layer for Relay by shipping presence APIs and typing broadcasts for channels, DMs, and threads.

## Added By Codex

- `GET /api/v1/presence`
- `POST /api/v1/presence`
- `POST /api/v1/typing`

## Realtime Behavior

- `POST /api/v1/presence` broadcasts `presence.updated`
- `POST /api/v1/typing` broadcasts `typing.updated`

## Contract Notes

- presence reuses the existing `users.status` field
- typing payload includes:
  - `user_id`
  - optional `channel_id`
  - optional `thread_id`
  - optional `dm_id`
  - `is_typing`

## Gemini Handoff

- hydrate avatar/profile status from `GET /api/v1/presence`
- consume `presence.updated` to keep status fresh without full refetch
- consume `typing.updated` to render transient typing indicators in:
  - channels
  - DMs
  - threads
