# v0.5.19 Release Notes

## Summary

This release adds the first real artifact lifecycle layer for Relay so the canvas surface can move from static demo content to persisted, AI-generatable collaborative outputs.

## Added By Codex

- `GET /api/v1/artifacts`
- `POST /api/v1/artifacts`
- `GET /api/v1/artifacts/:id`
- `PATCH /api/v1/artifacts/:id`
- `POST /api/v1/ai/canvas/generate`

## Realtime

- artifact mutations now broadcast `artifact.updated`

## Persistence

- added `artifacts`
- artifacts store:
  - channel ownership
  - title
  - type
  - status
  - content
  - source
  - provider/model
  - created/updated user ids

## Additional Hardening

- activity feed reaction items now use stable unique IDs, so repeated emoji reactions do not collide in inbox or activity surfaces

## Gemini Handoff

- hydrate canvas with `GET /api/v1/artifacts?channel_id=...`
- use `GET /api/v1/artifacts/:id` for detail hydration
- use `POST /api/v1/artifacts` for manual canvas creation
- use `PATCH /api/v1/artifacts/:id` for save/update
- use `POST /api/v1/ai/canvas/generate` for AI-generated artifacts
- listen for `artifact.updated` on websocket for live canvas refresh
