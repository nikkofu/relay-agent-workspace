# v0.6.20 Release Notes

Release date: 2026-04-23

## Summary

Codex Phase 63E makes Entity Ask AI streamable and persistent. The entity detail page can now load previous Q&A history without invoking the LLM and can render long answers progressively over SSE.

## API Changes

- `POST /api/v1/knowledge/entities/:id/ask`
  - Backward compatible response.
  - Now persists each successful answer to entity Ask history.
- `POST /api/v1/knowledge/entities/:id/ask/stream`
  - New SSE endpoint.
  - Emits `start`, `answer.delta`, `answer.done`, `done`, and `error`.
  - Persists the final answer on successful completion.
- `GET /api/v1/knowledge/entities/:id/ask/history`
  - Returns `{ entity, items[] }` for the current user.
  - Items are ordered newest first.

## Windsurf Handoff

- Hydrate entity Ask AI history on entity detail page load.
- Use stream ask for long answers and progressively render `answer.delta`.
- Append `answer.done.answer` to the visible Q&A list and keep sync ask as fallback.
