# v0.5.59

Date: 2026-04-21

## Summary

`v0.5.59` ships Phase 32: Operational Shell Controls. It adds the backend contracts needed for workflow run logs and deletion, richer file preview metadata, and channel-level notification/leave controls.

## Added APIs

- `GET /api/v1/workflows/runs/:id/logs`
  - Returns ordered execution logs for a workflow run.
  - Fields include `id`, `level`, `message`, parsed `metadata`, and `created_at`.
- `DELETE /api/v1/workflows/runs/:id`
  - Deletes a workflow run and its associated steps/logs.
  - Broadcasts `workflow.run.deleted` over WebSocket.
- `GET /api/v1/files/:id/preview`
  - Returns preview/download metadata for file detail and preview surfaces.
  - Supports `image`, `pdf`, and fallback `file` preview kinds.
- `GET /api/v1/channels/:id/preferences`
  - Returns the current user's channel notification preferences.
- `PATCH /api/v1/channels/:id/preferences`
  - Persists `notification_level` and `is_muted`.
  - Valid notification levels are `all`, `mentions`, and `none`.
- `POST /api/v1/channels/:id/leave`
  - Removes the current user from channel membership.
  - Recomputes channel `member_count`.

## Data Model

- Added `WorkflowRunLog`.
- Added `ChannelPreference`.
- Seed data now includes sample workflow logs and channel preferences.
