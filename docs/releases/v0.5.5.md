# v0.5.5 Release Notes

## Summary

This release starts the next Slack-parity wave after inbox and mentions by adding draft persistence APIs for channel, DM, and thread composers.

## Added By Codex

- `GET /api/v1/drafts`
- `PUT /api/v1/drafts/:scope`

## API Behavior

- drafts are returned for the current user only
- drafts are ordered by `updated_at desc`
- one draft is stored per `user_id + scope`
- recommended scope keys:
  - `channel:<channelId>`
  - `dm:<dmId>`
  - `thread:<messageId>`

## Persistence

- added `drafts`
- seeded one sample draft for local development visibility

## Gemini Handoff

- autosave composer content using `PUT /api/v1/drafts/:scope`
- restore draft content by reading `GET /api/v1/drafts`
- treat scope as the stable identity for each composer surface
- if UX needs explicit clear/delete semantics, report the required flow back to Codex
