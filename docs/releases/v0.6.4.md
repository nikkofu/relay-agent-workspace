# v0.6.4 - Phase 59 Knowledge Ops APIs

This release closes the backend gap behind Windsurf's `v0.6.3` Following Hub. Relay's knowledge layer now has bulk follow controls, workspace-level alert tuning, entity activity history, and a trending feed that can surface what is accelerating across the workspace.

## Added APIs

- `PATCH /api/v1/users/me/knowledge/followed/bulk`
- `GET /api/v1/workspace/settings`
- `PATCH /api/v1/workspace/settings`
- `GET /api/v1/knowledge/entities/:id/activity`
- `GET /api/v1/knowledge/trending`

## Behavior

- Bulk follow updates accept `entity_ids[]` plus a single `notification_level`.
- Workspace settings currently expose:
  - `spike_threshold`
  - `spike_cooldown_minutes`
- Entity activity returns daily `buckets[]` for sparkline-style rendering.
- Trending returns ranked entities with `recent_ref_count`, `previous_ref_count`, `velocity_delta`, `related_channel_ids`, and `last_ref_at`.
- Spike-alert detection now reads persisted workspace settings, while preserving defaults of `3` and `360` minutes for older workspaces.

## Windsurf Handoff

- Replace Following Hub fan-out Mute All with `PATCH /api/v1/users/me/knowledge/followed/bulk`.
- Add workspace-level spike controls in Settings or Knowledge surfaces using `GET|PATCH /api/v1/workspace/settings`.
- Add entity activity sparklines using `GET /api/v1/knowledge/entities/:id/activity`.
- Add a Trending block to `/workspace/knowledge` and Home using `GET /api/v1/knowledge/trending`.
