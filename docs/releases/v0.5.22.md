# v0.5.22 Release Notes

## Summary

This release adds file asset APIs for Relay and tightens artifact responses so Canvas surfaces can show real editor identity and prepare for attachment-aware workflows.

## Added By Codex

- `POST /api/v1/files/upload`
- `GET /api/v1/files`
- `GET /api/v1/files/:id`
- `GET /api/v1/files/:id/content`

## Artifact Contract Upgrade

- artifact list, detail, create, update, and AI-generate responses now include:
  - `created_by_user`
  - `updated_by_user`
- websocket `artifact.updated` payloads use the same hydrated contract

## Persistence

- added `file_assets`
- local dev uploads are stored under `apps/api/uploads/`

## Gemini Handoff

- use multipart upload to `POST /api/v1/files/upload` with form field `file` and optional `channel_id`
- use `GET /api/v1/files?channel_id=...` to render channel-scoped file lists
- use `GET /api/v1/files/:id` for metadata and returned `url` for direct fetch/download
- Canvas and artifact UI can now display last editor identity using `updated_by_user`
