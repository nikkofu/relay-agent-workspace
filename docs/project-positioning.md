# Relay Agent Workspace Positioning

## Product Name

- Full name: `Relay Agent Workspace`
- Short name: `Relay`

## One-Line Positioning

Relay is an AI-native collaboration workspace for humans and agents.

## Product Thesis

Relay combines realtime messaging, shared context, AI chat, artifacts, and agent execution inside one workspace. Instead of treating AI as an isolated assistant, Relay treats agents as participants in the same operating environment as people, channels, threads, and documents.

## What The Product Is

- A workspace-centered collaboration product
- A messaging-first interface for human and agent coordination
- A foundation for agent execution, tool calling, and artifact generation
- A TypeScript-first frontend that will connect to a database-backed, realtime backend

## What The Product Is Not

- Not a clone-only messaging demo
- Not just an AI sidebar on top of chat
- Not only a generic LLM UI shell
- Not yet a full production backend platform

## Core Concepts

### Workspace

The top-level shared environment containing members, channels, artifacts, and agent activity.

### Messaging

Channels, DMs, threads, reactions, and activity remain the primary coordination surface.

### Agents

Agents are treated as first-class collaborators that can respond, execute tasks, and produce outputs inside the workspace.

### Artifacts

Artifacts are durable AI-generated or human-edited outputs such as docs, plans, summaries, and structured work products.

### Orchestration

The system should evolve from simple AI responses to visible, auditable, multi-step agent execution.

## Target Users

- AI-native product and engineering teams
- Teams coordinating humans and autonomous or semi-autonomous agents
- Builders who want messaging, artifact creation, and agent workflows in one interface
